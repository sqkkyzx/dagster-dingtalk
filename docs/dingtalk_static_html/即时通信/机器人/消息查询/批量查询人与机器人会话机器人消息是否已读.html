批量查询人与机器人会话机器人消息是否已读

调用本接口，批量查询人与机器人会话时，机器人消息是否已读，适用于企业内部沟通中24小时内需要确认消息是否被阅读的场景。

接口调用说明

  • 一次最多可以查询一条单聊信息20名接收者的消息是否已读数据。

  • 若直接发起http请求，需要先对processQueryKey参数进行URLEncoding转义，然后再将转义后的processQueryKey拼入URL中。

权限要求
