项目协同

本文介绍了钉钉群聊机器人项目协同和进度同步的使用场景。

基础用法—通过卡片推送任务进度

通过机器人推送任务进度到群内是一个非常基础的用法，而在钉钉上，则可以通过互动卡片来实现更好的排版和视觉呈现。

如下图所示，通过图文混排的方式，可以很清楚的展示出现阶段性的进度和相关信息，实现项目协同的透明化。

基础用法—通过互动卡片分配任务给群成员

在很多项目的推进过程中需要分配不同的任务给到不同的人，那么通过群机器人推送互动卡片的方式，可以实现清晰的任务分配流程。

如图所示，在卡片中可以实现@人的效果，并具体指明谁负责哪个任务，对内一步了然。同时，卡片底部的查看详情按钮，也可以跳转进入具体页面查看详细情况。

高阶用法—响应式实时互动

如果利用互动卡片的实时刷新能力，则在任务分配的过程中，可以实现实时响应的效果。

如下图所示，当某个任务被人领取后，可以即时刷新卡片，告知群内其他人谁已经确认接受了某个任务。

高阶用法—确认关键信息已收到

如下图所示，当需要推送某个关键信息时候，我们会希望第一时间确定这条信息是否已经被目标用户读取了。因此，在推送某些关键信息时，可以在信息卡片下方放置一个确认收到的按钮，并要求所有人在已读信息后点击确认收到。这样，推送信息的业务系统就能实时统计和回收哪些人已读信息，哪些人未读信息，并作出针对性推送。

高阶用法—通过定向可见的消息进行二次提醒

在很多协同场景，因为信息量比较大，部分用户可能会漏掉一些信息，例如任务派送后没有第一时间看到导致错过了消息。

那么结合前述的方案，我们可以更进一步，在用推送全员可见的提醒消息之后，如果目标用户没有第一时间确认收到，则进一步在群内定向推送一条仅限这个用户可见的提醒消息，告知用户要尽快接收任务并作出响应。此时如果用户在这条定向可见的消息上确认接收，则之前的全员可见的卡片上也可以实时刷新内容告知某人已确认。

另一种情况下，例如某个用户创建了一个任务分配给了群内多个同事，那么也可以利用定向可见消息的逻辑，实现进度同步。例如任务创建1个小时后，给创建者推送一个仅限他可见的卡片消息，告知任务已经被所有人确认收到。
