群助手发送消息

本示例介绍了如何发送群助手消息。

本文档展示了，创建一个企业内部应用，使用场景群提供的API，实现群助手发送消息流程：

  1. 选择目标应用，进入应用详情页，单击基础信息 > 凭证与基础信息。

  2. 获取应用 Client ID 和 Client Secret。

  3. 申请发送群助手消息接口权限。

  4. 获取应用访问凭证，获取企业内部应用的accessToken。

  5. 在开发者后台-场景群页面，申请机器人，审核通过后获取机器人id参数robotCode。

  6. 在开发者后台-场景群页面，创建群模板，获取群模板id参数template_id。

  7. 绑定场景群-机器人到场景群-群模板，保存编辑，返回群场景列表。

  8. 进入对应群模板，提交审核，点击发布。

  9. 依据template_id调用创建群接口，获取openConversationId。

  10. 调用发送群助手消息接口，实现机器人群消息的发送。

前提条件

完成创建应用的流程。

操作步骤

  1. 选择目标应用，进入应用详情页，单击基础信息 > 凭证与基础信息。

  2. 获取应用 Client ID 和 Client Secret。

  3. 单击开发配置 > 权限管理，在权限搜索框中输入qyapi_chat_manage，并申请权限。

  4. 获取应用访问凭证获取企业内部应用的accessToken。调用接口时，通过accessToken鉴权调用者身份。

  5. 在开发者后台-场景群页面，申请机器人，获取机器人ID参数，即robotCode。

    说明

    申请机器人需要审核，审批通过后可以查看ID，如下图所示。

  6. 在开发者后台-场景群页面，创建群模板，获取群模板id参数template_id。

  7. 绑定场景群-机器人到场景群-群模板，保存编辑，返回群场景列表。

  8. 进入对应群模板，进行提交审核，点击发布。

    说明

    审核自动通过，无需审批。

  9. 通过template_id调用创建群接口，获取openConversationId。

  10. 调用发送群助手消息接口，实现机器人群消息的发送。
