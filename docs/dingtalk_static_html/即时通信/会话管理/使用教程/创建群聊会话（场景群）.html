创建群聊会话（场景群）

本操作流程指导你如何实现创建群聊会话且存量群升级为群聊会话。

步骤一：创建钉钉应用

  1. 登录开发者后台。

    重要

    如果你无法使用企业组织登录开发者后台，请参考成为钉钉开发者文档，确保已有开发者权限。

  2. 单击应用开发，创建一个企业内部应用或第三方企业应用，应用类型可以选择小程序或H5微应用。以创建企业内部应用为例。

说明

使用群聊会话相关能力，必须先创建企业内部应用或第三方企业应用，获取相关的访问凭证。

3. 以创建企业内部应用为例，创建应用后，可以获取AppKey和AppSecret。

说明

用于后续获取接口调用的访问凭证，详见本文第5步使用。

4. 添加接口调用权限。搜索“场景群”，申请场景群接口对应的权限。

说明

用于后续步骤正常调用群聊会话相关接口，未申请接口权限，调用接口时会提示“没有调用该接口的权限”。

5. 获取应用访问凭证。以企业内部应用为例，调用获取企业内部应用的access_token。调用群聊会话相关接口时，通过accessToken鉴权调用者身份。

说明

访问凭证用于在调用群聊会话开放相关接口时使用。

步骤二：创建群模板

  1. 在开发者后台，单击场景群，然后选择群模板，单击创建群模板。

  2. 选择应用类型，然后选择一个已创建的钉钉应用，单击下一步。

  3. 配置群模板信息，然后单击保存编辑。

    群模板信息项

    是否必填

    填写说明

    群模板名称

    必填

    输入群模板的名称。

    图标

    必填

    选择群模板的图标。

    描述

    必填

    输入该模板的描述信息。

    群设置

    非必填

    设置群相关内容。

4. 在群模板列表页面，单击新创建的群模板。

5. 点击提交审核。

说明

提交审核后，审批自动通过。

6. 点击发布。

7. 发布成功后，查看群模板ID。

步骤三：创建群聊会话

通过接口创建群聊会话

  1. 调用创建群接口创建群聊会话。

    重要

    第三方企业应用创建群接口前，需要设置灰度企业。

    DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/im/chat/scenegroup/create");
OapiImChatScenegroupCreateRequest req = new OapiImChatScenegroupCreateRequest();
req.setOwnerUserId("manager4220");
req.setUserIds("user01,user02");

req.setShowHistoryType(0L);
req.setValidationType(0L);
req.setSearchable(0L);

OapiImChatScenegroupCreateResponse rsp = client.execute(req, access_token);
System.out.println(rsp.getBody());

存量群升级群聊会话

存量群升级群聊会话有2种方式：

方式一：钉钉群里，在群设置中启动群模板

方式二：调用启用群模板API升级。

哪些群可以升级成群聊会话

目前群聊会话主要服务具有企业归属群，企业内部群、服务群是具有企业归属的，目前这两类群可以通过API升级为群聊会话。

升级条件

在将企业内部群、服务群升级为群聊会话前，确保满足以下条件：

  1. 该群模板已经灰度到对应群的企业。

  2. 群模板需要群主或群管理员身份才可以启用/停用。

  3. 第三方企业应用群模板还需要效验目标群对应的企业是否有安装群模板对应的应用。

方式一：在群设置中启动群模板

方式二：通过调用API启用群模板

调用启用群模板接口，实现群升级为群聊会话。

重要

调用启用群模板接口前，需要设置灰度企业。
