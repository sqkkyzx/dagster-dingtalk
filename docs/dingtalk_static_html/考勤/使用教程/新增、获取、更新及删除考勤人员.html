新增、获取、更新及删除考勤人员

本文介绍了创建一个企业内部应用，使用考勤人员管理提供的API，实现新增、获取、更新和删除考勤人员等。

接入流程简介

前提条件：完成创建应用的流程

步骤一：获取应用凭证信息，获取应用 Client ID 和 Client Secret。

步骤二：申请接口权限，申请考勤相关接口权限。

步骤三：获取应用访问凭证获取企业内部应用的access_token。调用接口时，通过accessToken鉴权调用者身份。

步骤四：调用考勤相关API：

  1. 调用服务端API-批量新增参与考勤人员接口，实现新增考勤组参与考勤人员。

  2. 调用服务端API-获取用户考勤组，获取用户所在考勤组信息。

  3. 获取考勤组内参与考勤人员信息

    • 调用服务端API-获取参与考勤人员的userid接口，获取参与考勤人员的userId信息。

    • 调用服务端API-获取参与考勤人员接口，获取参与考勤人员信息。

    • 调用服务端API-查询参与考勤人员列表，获取参与考勤人员列表信息。

  4. 调用服务端API-校验用户是否在当前考勤组接口，实现校验某个部门或者员工，是否在某考勤组内。

  5. 调用服务端API-更新参与考勤人员接口，实现更新考勤组参与考勤人员。

  6. 调用服务端API-批量删除参与考勤人员接口，实现删除参与考勤人员。

前提条件

完成应用创建与配置的流程。

步骤一：获取应用凭证

  1. 选择目标应用，进入应用详情页，单击基础信息 > 凭证与基础信息。

  2. 获取应用 Client ID 和 Client Secret。

步骤二：添加接口权限

单击开发配置 > 权限管理，在权限搜索框中输入qyapi_attendance_group_manage和qyapi_attendance_group_read，并申请权限。

步骤三：获取应用访问凭证accessToken

重要

服务端API差异详情参见旧版API VS 新版API。以下接口均使用服务端API接口，SDK下载详情参见服务端SDK下载。

根据步骤一中 的 Client ID 和 Client Secret，获取应用访问凭证获取企业内部应用的access_token。
