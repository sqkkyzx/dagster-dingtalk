覆写文档（应用授权）

调用该接口，可以通过Markdown的格式覆写整篇文档内容。

请求

基本信息

HTTP URL

https://api.dingtalk.com/v1.0/doc/suites/documents/{docKey}/overwriteContent

HTTP Method

POST

支持的应用类型

appType-企业内部应用

权限要求

permission-Storage.File.Write-企业存储文件写权限

请求头

名称

类型

是否必填

描述

x-acs-dingtalk-access-token

String

是

调用该接口的访问凭证，通过调用获取企业内部应用的accessToken接口获取。

查询参数

名称

类型

是否必填

描述

operatorId

String

是

用户unionId，可通过查询用户详情接口获取unionid参数值。

路径参数

名称

类型

是否必填

描述

docKey

String

是

文档ID（文档的 docKey 或 dentryUuid），通过创建知识库文档接口创建文档后从返回值中获取docKey参数。

说明

从文档URL获取dentryUuid，以https://alidocs.dingtalk.com/i/nodes/Z9xxxxa-id为例，dentryUuid 就是Z9xxxxa-id。

请求体

名称

类型

是否必填

描述

dataType

String

否

覆写内容的数据格式，取值：

  • markdown：待覆写内容为 markdown 格式，默认值。

content

String

是

需要写入的内容字符串，注意：

  • 若 dataType 为 markdown，则这里为 markdown 格式的字符串。

  • 最大长度为 50000 字符。

POST /v1.0/doc/suites/documents/someDocKey/overwriteContent?operatorId=someOperatorId HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:7dc9ec4xxx
Content-Type:application/json

{
  "dataType" : "markdown",
  "content" : "# heading"
}
