自定义快捷审批：支持在待办中心、OA审批消息卡片快捷审批来源于三方业务系统的待处理任务

场景介绍

适用场景

客户希望把多个三方系统的审批任务都集中在钉钉审批中心，做一站式沉浸审批。同时希望支持三方系统配置审批任务的回调接口，实现在钉钉审批待办中心、OA审批消息卡片上直接进行快捷审批操作，实现一键审批。为用户、特别是审批事项繁多的管理层带来超级体验。

业务流程

实现效果

支持在待办中心列表页、OA审批消息卡片快捷审批来源于三方业务系统的待处理任务。

开发流程

以企业开发一个内部应用实现能力集成举例，实现三方业务系统发起，钉钉端内打开业务系统详情页进行审批，并在待办中心列表页、OA审批消息卡片快捷审批来源于三方业务系统的待处理任务。具体实现流程如下图所示：（注意此场景下，若只需要自定义快捷审批按钮，不需要自定义审批业务分组，则步骤1-5业务分组相关的流程都不涉及）

整体链路

详细流程

接入流程简介

本文档展示了，创建一个企业内部应用，使用OA审批流程中心提供的API，通过创建/删除业务分组、创建/更新/删除三方审批模板、创建/更新审批实例、创建/更新/查询审批待办任务、清理OA审批数据等API，实现业务系统发起，钉钉端内打开业务系统详情页进行集成，并支持自定义快捷审批按钮的场景案例。

前提条件：完成创建应用的流程。

步骤一：进入应用详情页，获取应用 Client ID 和 Client Secret。

步骤二：申请接口权限，申请“OA审批”相应权限。

步骤三：获取应用访问凭证获取企业内部应用的accessToken。调用接口时，通过accessToken鉴权调用者身份。

步骤四：调用OA审批相关API：

  1. 注册自定义快捷审批按钮callback回调接口：用于审批人在钉钉待办中心、OA审批消息卡片上直接进行审批操作时，钉钉会回调三方系统，三方系统收到回调后更新任务信息，并将新的任务信息同步回钉钉审批中心，形成闭环。网关回调钉钉外数据接口需要统一在“数据源管理”中注册成网关，详细的使用说明请参考创建数据源。

  2. 同步审批模板数据到钉钉：调用新版服务端API-保存流程中心外部集成审批模板接口，获取模板的唯一编码processCode。注意：该场景下无需进行processFeatureConfig流程中心集成配置，请求示例如下：

{
    "name": "快捷审批",
    "description": "保存流程中心外部集成审批模板 (高级版专享接口)",
    "formComponents": [
        {
            "componentType": "TextField",
            "props": {
                "componentId": "TextField-1",
                "label": "单行输入框",
                "required": true,
                "bizAlias": "TextField-bizAlias",
                "disabled": false
            }
        }
    ],
    "templateConfig": {
        "disableSendCard": false
    }
}

  1. 如果没有保存processCode，可以通过调用获取模板code接口获取processCode。

  2. 创建审批模板成功后，用户可以在钉钉OA审批管理后台，查看三方自有审批单模板、查看/搜索模板数据、导出/删除模板数据等。

  3. 同步审批实例数据到钉钉：根据模板编码processCode，调用新版服务端API-保存流程中心外部集成审批实例接口发起审批实例，获取审批实例processInstanceId。注意：该场景下无需进行featureConfig流程中心集成配置 ，请求示例如下：

    {
    "formComponentValueList": [
        {
            "name": "单行输入框",
            "value": "[钉钉](https://www.dingtalk.com/)"
        }
    ],
    "title": "快捷审批",
    "processCode": "PROC-C818F3C2-BDAB-4055-847A-CAF1EB56E1A8",
    "originatorUserId": "1451694214729725262",
    "url": "http://www.dingtalk.com",
    "notifiers": [
        {
            "userid": "1451694214729725262",
            "position": "start"
        }
    ]
}

  4. 创建审批实例成功后，用户也可以进入钉钉OA审批中心，查看审批四大列表（待处理、已处理、已发起、我收到的）、搜索审批实例数据、执行审批等操作。

  5. 同步审批任务数据到钉钉：根据审批实例processInstanceId和待办事项列表tasks，调用保存流程中心外部集成审批任务接口，可以将三方系统内的审批节点信息同步到钉钉OA审批，获取待办事项的taskId并生成对应的钉钉待办任务。同时三方系统可以通过featureConfig配置审批任务的回调接口，这样审批人可以在钉钉待办中心、OA审批消息卡片直接进行审批操作。注意：callback中的apiKey是在步骤1中提前注册好的数据源。

{
    "processInstanceId": "OW3kMrjfT9Crje23kP-C3g06701727322896",
    "activityId": "xxx-1",
    "tasks": [
        {
            "userId": "1451694214729725262",
            "url": "https://www.dingtalk.com"
        }
    ],
    "featureConfig": {
        "features": [
            {
                "name": "CUSTOM_SHORTCUT",
                "runType": "REDIRECT",
                "callback": {
                    "apiKey": "applyIntegratedCustomAction",
                    "appUuid": "dingec6f7227666ad00ba39a90f97fcb1e09",
                    "version": "1"
                }
            }
        ]
    }
}

  1. 创建待处理任务成功后实现效果如下。调用查询通过流程中心集成的OA审批任务接口，可以查询到用户运行中的审批任务。

  1. 同步审批任务状态到钉钉：根据审批实例processInstanceId和审批待办任务taskId，可以调用更新流程中心任务状态接口，同步完成自有审批待办状态的更新。在或签等场景，可以调用批量取消流程中心待处理任务接口，批量将审批实例下正在运行中的待办事项设置为CANCELED。

  2. 同步审批实例状态到钉钉：根据审批实例processInstanceId和实例状态status、实例结果result等，可以调用更新实例状态或 批量更新实例状态接口，更新实例状态。

  3. 最后，若需要对审批模板数据进行清理，可以调用删除模板接口，删除为企业创建的审批模板，同时删除该模板下创建的实例和待办任务。
